# Shopo Alpha Agent - MVP 开发计划

## 产品定位

**核心价值：** AI 友好的 Agent 聚合展示平台，为 AI 搜索引擎（ChatGPT、Claude、Perplexity）提供结构化的 Agent 信息。

**目标用户：**
- 主要：AI 搜索引擎（通过 GEO 优化）
- 次要：通过 AI 推荐来的人类用户

**差异化：**
- 专注 Agent（不是通用 AI 工具）
- 深度分析（不只是列表）
- AI 优先（GEO > SEO）

## MVP 功能范围

### ✅ 核心功能（必须有）

#### 1. Agent 展示系统
- [x] Agent 列表页（分类筛选）

- [ ] 搜索功能（基础文本搜索）

## 技术架构

### 技术栈选择理由

| 技术 | 选择理由 | 替代方案 |
|------|---------|---------|
| **Next.js 14** | SSG 支持好，SEO 优化，部署简单 | Astro, Remix |
| **Supabase** | 一站式解决方案（DB + Auth + API） | Firebase, PlanetScale |
| **Tailwind CSS** | 快速开发，响应式 | Bootstrap, Chakra UI |
| **Playwright** | 支持 JS 渲染，稳定 | Puppeteer, Cheerio |
| **OpenAI GPT-4** | 分析质量高 | Claude, Gemini |
| **Vercel** | 免费，自动 CI/CD | Netlify, Railway |

### 数据流

```
爬虫源（GPT Store, Poe）
    ↓
Playwright 爬取原始数据
    ↓
GPT-4 分析并结构化
    ↓
Supabase PostgreSQL 存储
    ↓
Next.js SSG 生成静态页面
    ↓
Vercel CDN 全球分发
    ↓
AI 搜索引擎爬取
    ↓
用户通过 AI 发现
```

## 风险和应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 爬虫被封 | 中 | 高 | 使用代理，降低频率，准备种子数据 |
| OpenAI 成本过高 | 中 | 中 | 缓存结果，批量处理，考虑更便宜的模型 |
| AI 不引用内容 | 高 | 高 | 持续优化 GEO，增加内容质量 |
| 用户增长慢 | 中 | 中 | 内容营销，社区运营，SEO 优化 |
| 竞争对手出现 | 高 | 中 | 快速迭代，建立社区，专注垂直领域 |

## 总结

这个 MVP 设计遵循以下原则：

1. **最小可行**：只包含核心功能，快速验证
2. **技术简单**：使用成熟技术栈，降低风险
3. **成本可控**：免费额度足够 MVP 阶段
4. **快速迭代**：2-3 周上线，根据反馈优化
5. **AI 优先**：专注 GEO，而不是传统 SEO

**关键成功因素：**
- 内容质量 > 数量
- GEO 优化到位
- 快速上线，快速迭代
- 持续监控 AI 搜索效果
