## 需求背景
需要在现有项目中新增一个「AI 数字资产价值索引」类的着陆页，整体参考提供的设计截图，保持与现有 Tailwind/Next App Router 架构一致（`app/` 路由、`components/` 里复用 UI 组件、渐变背景风格与现有首页一致）。

## 需求 1：顶部导航与英雄区
1. 页面位置与骨架  
   - 在 `app/` 下新增对应路由（例如 `app/agents/value-index/page.tsx`，文件命名保持 kebab-case），使用服务端组件，必要的交互用客户端子组件。  
   - 通过 `app/layout.tsx` 已注入的全局样式与 Tailwind 变量，保持深色渐变背景、圆角卡片和玻璃拟态感。
2. 顶部导航  
   - 左侧 Logo/标题（示例：`GEO 框组`），中间二级导航（`Agent 市场`、`探索 Agent`），右侧语言切换（`EN`/`中`）按钮样式与全局按钮风格一致。  
   - 导航在滚动时保持固定或留出足够内边距，确保与英雄区对齐。
3. 英雄主标题区  
   - 大标题「AI 时代的数字资产」「价值索引。」两行，突出颜色对比（白 + 品牌色）。  
   - 副标题简述：“链接 AI 时代的数字资产，全网 Agent 流量风向标。” 支持多行展示。  
   - 核心标签条：包含“实时报”“GEO 数据”“语义结构化分析”等高亮标签，使用统一的 pill 样式。  
   - 在标题下提供轻量 CTA（例如“立即查看榜单”按钮）以引导到下方榜单区。
4. 受众卡片（四列）  
   - 卡片示例：开发者 / 品牌运营 / 用户 / 产品经理；每张卡片包含图标、标题、两行描述。  
   - 使用统一卡片组件（可复用 `components/ui/card` 或现有卡片组件），暗色底、圆角、hover 提升。

## 需求 2：热门风向标榜单区
1. 榜单标题与筛选  
   - 标题「热门风向标」，右侧展示活跃榜单数量（示例“3 活跃”）。  
   - 搜索框：占位提示“搜索能力或标签…”，支持本地过滤列表（按名称、标签、能力关键词）。  
   - 预留 tabs/筛选（如按时间、来源模型）的位置，样式参考导航标签。
2. 榜单卡片列表（横向卡片流）  
   - 每张卡片展示：封面图、名称、类型标签（Chat/Task/Voice 等）、总热度数值及环比箭头、核心描述。  
   - “最大流量源”条形图：多色分段（按模型来源如 Claude/Gemini），右侧显示具体数值；分段比例可由数据驱动。  
   - 标签组：例如 LEGAL、COMPLIANCE；保证溢出时自动换行或截断。  
   - 排名徽标（#1、#2、#3）和角标位置与设计一致，支持 hover 阴影。  
   - 支持横向滚动或自适应三列栅格；移动端降为单列或两列。
3. 数据与状态  
   - 数据来源：优先复用 Supabase 现有 Agent 表或 mock 数据，字段至少包含：`id`, `name`, `type`, `heatScore`, `heatChange`, `summary`, `trafficSources[]`（包含来源名、数值、颜色）、`tags[]`, `coverImage`, `rank`.  
   - 需要加载态骨架（skeleton）和空状态提示（无匹配 Agent 时）。  
   - 交互：搜索实时过滤，点击卡片可跳转到 Agent 详情（可复用现有 `/agents/[id]` 路由或占位链接）。

## 验收标准
- 新增路由可通过 `npm run dev` 访问，视觉布局与截图接近（深色渐变背景、卡片层次、品牌色强调）。  
- 导航、英雄区、四张受众卡片完整渲染；榜单区包含标题、搜索、至少三张示例卡片（支持 mock 数据）。  
- 组件使用现有 `@/components` 或新增的复用型 UI 组件，样式优先 Tailwind Utility。  
- 具备移动端断点适配（排版缩放、栅格折行、滚动不溢出）。  
- 无 ESLint/TypeScript 报错，页面构建通过 `npm run lint`。
